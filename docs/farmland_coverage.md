# Покрытие таблицы `agro_farmland` по субъектам РФ

Вектор сельскохозяйственных угодий (а **не** данные Росреестра).
Таблица наполняется командой `import_farmlands_rosreestr` (название
историческое — команда просто принимает shp-архивы по ЗСН 5.3,
источник данных может быть любой).

Последняя заливка: **19 638 762 полигонов по 73 субъектам РФ.**

## Субъекты без данных

Следующие регионы существуют в таблице `agro_region`, но **не покрыты
вектором угодий** — исходных shp-файлов для них нет:

- Владимирская область
- Ивановская область
- Костромская область
- Липецкая область
- Смоленская область
- Тульская область
- Томская область
- Республика Марий Эл
- Республика Хакасия
- Еврейская автономная область

Для этих регионов карта угодий будет пустой до тех пор, пока не
появится источник shp-данных.

## Субъекты без с/х угодий (по определению)

Города федерального значения — в них нет сельхозугодий как класса:

- Москва
- Санкт-Петербург

Эти регионы присутствуют в `agro_region` для полноты справочника,
но `agro_farmland` по ним заведомо пустой.

## Как обновлять покрытие

1. Положить shp-архив для нужного региона в
   `/opt/edunabazar/import_data/<Имя региона>/` (распознаётся по имени
   папки, соответствующему `Region.name` — см. резолвер в
   `agrocosmos/management/commands/import_farmlands_rosreestr.py`).
2. Запустить:
   ```bash
   docker compose -f docker-compose.prod.yml exec -T web \
     python manage.py import_farmlands_rosreestr \
       --base /data/import \
       --region-dir "<Имя папки>"
   ```
3. Проверить счётчик:
   ```sql
   SELECT COUNT(*) FROM agro_farmland
   WHERE region_id = (SELECT id FROM agro_region WHERE name = '<регион>');
   ```

Обновить этот список при добавлении/удалении покрытия.
