# Агрокосмос — Техническое описание улучшений мониторинга

## Часть 1: MODIS (250м)

### Текущее состояние
- Terra + Aqua merge → SummaryQA ≤ 1 → 16-дневный median → зональная статистика
- min_valid = 0.50

### 1.1. Временна́я интерполяция (gap-filling)
**Проблема**: при сплошной облачности композит содержит NaN. На графике — пропуски.

**Решение**: после расчёта зональной статистики по всем композитам — интерполяция временного ряда (Savitzky-Golay или Whittaker smoother). Это стандарт в продуктах TIMESAT и NASA VNP13.

```
Сырые:   0.45, NaN, NaN, 0.52, 0.61, NaN, 0.58
Сглажен: 0.45, 0.48, 0.50, 0.52, 0.61, 0.60, 0.58
```

**Реализация**: постобработка VegetationIndex записей через scipy.signal.savgol_filter + новое поле mean_smooth.

**Статус**: [x] Реализовано — команда `ndvi_postprocess`, поле `VegetationIndex.mean_smooth`

---

### 1.2. Взвешивание по площади при агрегации на район
**Проблема**: сейчас Avg('mean') — простое среднее. Поле 1 га и 500 га вносят одинаковый вклад.

**Решение**: Sum(F('mean') * F('farmland__area_ha')) / Sum('farmland__area_ha') — средневзвешенное по площади. Это стандарт FAO и Copernicus Global Land Service.

**Статус**: [x] Реализовано — `views.py:api_ndvi_stats` (by_crop, by_period, summary)

---

### 1.3. Фильтрация аномалий (spike detection)
**Проблема**: даже при SummaryQA ≤ 1 бывают выбросы — резкое падение NDVI из-за остаточной дымки или тумана.

**Решение**: правило 3σ-отклонения от скользящего среднего или best-index slope threshold (BISE) — если NDVI упал более чем на 0.15 за один шаг, а потом вернулся, это артефакт. Помечать is_anomaly = True, не включать в агрегацию.

**Статус**: [x] Реализовано — команда `ndvi_postprocess`, поле `VegetationIndex.is_anomaly`, API исключает аномалии

---

### 1.4. Фенологические метрики
**Практика**: USDA CropScape, Copernicus HR-VPP.

Из сглаженного ряда NDVI извлекать:
- **SOS** (Start of Season) — дата начала вегетации
- **EOS** (End of Season) — конец вегетации
- **POS** (Peak of Season) — дата максимума NDVI
- **Max NDVI** — пиковое значение
- **LOS** (Length of Season) — длительность в днях
- **TI** (Total Integrated NDVI) — интеграл под кривой ≈ суммарная продуктивность

Хранить в отдельной таблице FarmlandPhenology (farmland, year, sos, eos, pos, max_ndvi, los, ti).

**Статус**: [x] Реализовано — модель `FarmlandPhenology`, команда `compute_phenology`

---

### 1.5. Baseline / аномалия года
**Проблема**: один год NDVI ничего не говорит без контекста.

**Решение**: хранить многолетний baseline (среднее за 5+ лет) и вычислять z-score аномалии:

```
anomaly = (NDVI_current - NDVI_baseline) / σ_baseline
```

Отображать на карте: красный (засуха/деградация) → зелёный (норма) → синий (выше нормы). Это подход FEWS NET и NASA GIMMS.

**Статус**: [x] Реализовано — `NdviBaseline.std_ndvi`, z-score в API `by_period`, `calc_ndvi_baseline` считает STDDEV

---

### 1.6. Улучшенная QA-фильтрация
**Сейчас**: SummaryQA ≤ 1 (good + marginal).

**Улучшение**: дополнительно использовать биты DetailedQA:
- Bit 0-1: VI Quality (только 00 = highest)
- Bit 6-7: Aerosol quantity (отсеять high aerosol = 11)

Это снижает шум в ~1.5 раза (Didan et al., 2015).

**Статус**: [x] Реализовано — `satellite_modis_raster.py:_process()` использует DetailedQA биты

## Часть 2: Sentinel-2 (10м) + Landsat 8/9 (30м)

### Текущее состояние
- S2: SCL маска (eq 4,5,6,7) → 5-дневный median → зональная статистика
- L8: QA_PIXEL (cloud + shadow) → 16-дневный median → гармонизация Roy et al. → зональная статистика
- min_valid = 0.70

### 2.1. Улучшенная облачная маска S2 — s2cloudless
**Проблема**: SCL маска Sentinel-2 пропускает ~10-15% тонких/полупрозрачных облаков и тени (Coluzzi et al., 2018).

**Решение**: использовать коллекцию COPERNICUS/S2_CLOUD_PROBABILITY (s2cloudless от Sentinel Hub) вместо SCL:

```python
s2_clouds = ee.ImageCollection('COPERNICUS/S2_CLOUD_PROBABILITY')
# Присоединить к S2, маскировать cloud_probability > 40%
```

Порог 40% — стандарт ESA; можно сделать настраиваемым. Даёт на ~30% меньше облачных артефактов.

**Статус**: [ ] Не реализовано

---

### 2.2. Маска теней для Landsat
**Сейчас**: только bit 3 (cloud) + bit 4 (cloud shadow).

**Улучшение**: добавить bit 1 (dilated cloud) + bit 5 (snow/ice):

```python
cloud_mask = (qa.bitwiseAnd(1 << 1).eq(0)    # dilated cloud
              .And(qa.bitwiseAnd(1 << 3).eq(0))  # cloud
              .And(qa.bitwiseAnd(1 << 4).eq(0))  # shadow
              .And(qa.bitwiseAnd(1 << 5).eq(0)))  # snow
```

**Статус**: [ ] Не реализовано

---

### 2.3. Объединённый S2+L8 композит (data fusion)
**Проблема**: S2 и L8 считаются отдельно. В облачный период оба могут дать пропуск.

**Решение**: после гармонизации L8 → S2-эквивалент объединять в единый временной ряд. При наличии обоих наблюдений за один период — брать средневзвешенное с весом пропорциональным количеству валидных пикселей:

```
NDVI_fused = (NDVI_s2 × N_valid_s2 + NDVI_l8 × N_valid_l8) / (N_valid_s2 + N_valid_l8)
```

Это подход HLS (Harmonized Landsat Sentinel) от NASA.

**Статус**: [x] Реализовано — команда `compute_fused_ndvi`, satellite=`hls_fused`, API `source=fused`

---

### 2.4. Дополнительные индексы
Модель уже поддерживает: EVI, MSAVI, NDWI, NDMI (поля в VegetationIndex).

Рекомендации:
- **EVI** — устойчивее NDVI при высокой биомассе (не насыщается), стандарт для тропических/субтропических культур
- **NDMI** ((B8A - B11) / (B8A + B11) у S2) — индекс влажности, ранний индикатор засухи
- **NDWI** — водный стресс, полезен для оценки орошения
- **MSAVI** — лучше NDVI на ранних стадиях вегетации (открытая почва)

**Статус**: [ ] Не реализовано

### 2.5. Попиксельный анализ внутри полигона
**Проблема**: сейчас хранятся только агрегированные значения (mean, median, min, max).

**Решение**: добавить гистограмму NDVI по полигону (5 бинов: 0-0.2, 0.2-0.4, 0.4-0.6, 0.6-0.8, 0.8-1.0). Это позволяет выявлять:
- Неоднородность поля (часть поля деградирует)
- Частичную гибель посевов (бимодальное распределение)

Хранить как JSON: histogram = [12, 8, 45, 130, 55] (количество пикселей по бинам).

**Статус**: [ ] Не реализовано

### 2.6. Детекция изменений (change detection)
**Практика**: Copernicus Land Monitoring, USDA NASS.

Автоматический алерт если:
- NDVI упал ≥ 0.2 за одну декаду (возможная гибель посевов)
- NDVI на ≥ 2σ ниже baseline (засуха)
- Поле не «зеленеет» к ожидаемой дате SOS ± 15 дней

Реализуется как Django management command + модель Alert.

**Статус**: [ ] Не реализовано

---

### 2.7. Валидация физических диапазонов
**Сейчас**: NDVI не фильтруется на физическую валидность.

**Решение**: на этапе зональной статистики отсекать:
- NDVI < -0.2 (вода/артефакт)
- NDVI > 1.0 (невозможно)
- Для с/х угодий: NDVI < 0.05 в вегетационный период → подозрительно (распашка? пожар?)

**Статус**: [ ] Не реализовано

---

### 2.8. Взвешивание «хороших» пикселей
**Сейчас**: все валидные пиксели полигона равноценны.

**Улучшение**: присваивать вес по quality score — пиксели близкие к краю облака или с высоким аэрозолем получают меньший вес. Подход из Landsat ARD (Analysis Ready Data).

**Статус**: [ ] Не реализовано
